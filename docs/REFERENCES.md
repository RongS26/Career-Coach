# References / 参考资料

## English

Use references as inspiration and source material, not as a script to copy. Keep licensing, attribution, and privacy constraints in mind.

### Career And Product Thinking

- Lenny's Newsletter and Lenny's Podcast: product leadership, growth, career development, and interview preparation.
- Marty Cagan, *Inspired* and *Transformed*: product teams, discovery, empowered teams.
- Shreyas Doshi's product management writing: product sense, execution, influence, career growth.
- Julie Zhuo, *The Making of a Manager*: management and feedback loops.
- Kim Scott, *Radical Candor*: feedback, coaching, and communication.
- Roger Martin, *Playing to Win*: strategy questions and tradeoff framing.
- Hamilton Helmer, *7 Powers*: strategic power and durable advantage.

### Coaching And Decision Frameworks

- Weekly review systems: goals, outcomes, blockers, next actions.
- Behavioral interview methods: STAR, CAR, impact-oriented storytelling.
- Negotiation frameworks: BATNA, reservation value, total compensation, risk-adjusted upside.
- Career capital framing: skills, reputation, network, leverage, and optionality.

### Technical Building Blocks

- Markdown context files for transparent, editable knowledge.
- Retrieval only for large reference sets; keep high-value coaching rules in prompt context.
- Local storage for private notes.
- Provider abstraction for OpenAI, Anthropic, DeepSeek, or other model APIs.
- Redaction checks before committing any context files.

### Reference Selection Rules

Choose references that are:

- Relevant to the user's target role
- Specific enough to guide decisions
- Non-confidential
- Easy to summarize into reusable principles
- Traceable back to source material

Avoid dumping huge content collections into the prompt. Summarize references into a compact, curated digest that the coach can actually use.

## 中文

Reference 应该作为启发和 source material，而不是可以直接照抄的脚本。使用时要注意授权、署名和隐私边界。

### 职业与产品思考

- Lenny's Newsletter 和 Lenny's Podcast：产品领导力、增长、职业发展和面试准备。
- Marty Cagan, *Inspired* 和 *Transformed*：产品团队、发现流程、empowered teams。
- Shreyas Doshi 的产品管理写作：product sense、execution、influence、career growth。
- Julie Zhuo, *The Making of a Manager*：管理与反馈闭环。
- Kim Scott, *Radical Candor*：反馈、coaching 和沟通。
- Roger Martin, *Playing to Win*：战略问题和取舍框架。
- Hamilton Helmer, *7 Powers*：战略势能和长期优势。

### Coaching 与决策框架

- Weekly review systems：目标、结果、阻塞点、下一步动作。
- Behavioral interview methods：STAR、CAR、impact-oriented storytelling。
- Negotiation frameworks：BATNA、reservation value、total compensation、risk-adjusted upside。
- Career capital framing：技能、声誉、人脉、杠杆和选择权。

### 技术构件

- 用 Markdown context files 保存透明、可编辑的知识。
- 只有 reference 很大时才使用 retrieval；高价值 coaching rules 应该放在 prompt context 里。
- 本地存储私有笔记。
- 对 OpenAI、Anthropic、DeepSeek 或其他模型 API 做 provider abstraction。
- 提交任何 context 文件前做 redaction check。

### Reference 选择规则

选择这样的 reference：

- 与用户目标岗位相关
- 足够具体，能指导决策
- 非机密
- 容易总结成可复用原则
- 可以追溯到来源

避免把大量内容直接塞进 prompt。应该整理成紧凑、精选、coach 真正能用的 digest。
